# Почтовый бот-автоответчик (Outlook → Supabase)

Бот следит за ящиком Outlook, распознаёт письма-заявки «прошу направить
реквизиты для оплаты курса», вытаскивает из них данные (курс, организация,
ИНН, КПП, адрес, ФИО слушателей), сохраняет их в таблицу Supabase и отвечает
отправителю шаблоном.

## Как устроен

| Файл | Что делает |
|---|---|
| `bot.js` | основной цикл: проверка почты, правила, ответы |
| `parser.js` | разбор текста заявки (метки, ИНН/КПП, ФИО) |
| `lib.js` | доступ к Microsoft Graph, обновление токенов, лог |
| `db.js` | запись заявок в Supabase (REST) |
| `login.js` | вход в Microsoft-аккаунт (device code flow) |
| `rules.json` | настройки: шаблон ответа, правила, интервалы |
| `schema.sql` | SQL для создания таблицы `payment_requests` |

Авторизация — через публичный client ID «Microsoft Graph Command Line Tools»,
поэтому регистрировать своё приложение в Azure не нужно (личным аккаунтам
Microsoft портал Azure недоступен).

## Запуск

```bash
# 1. вход в почту (один раз): получить код…
node login.js request
# …ввести его на https://www.microsoft.com/link и дождаться подтверждения
node login.js poll

# 2. доступ к базе: создать таблицу из schema.sql в Supabase SQL Editor,
#    затем положить рядом supabase.json:
#    { "url": "https://<проект>.supabase.co", "service_role_key": "sb_secret_..." }

# 3. проверка одним проходом / постоянная работа
node bot.js once
node bot.js
```

Секреты (`token.json`, `supabase.json`) хранятся только на сервере
и в репозиторий не попадают — см. `.gitignore`.

## Защита от лишних ответов

- одному отправителю — не чаще раза в сутки (`reply_cooldown_hours`);
- рассылки и no-reply адреса пропускаются (`skip_senders`);
- письма без признаков заявки остаются без ответа.
